       └────────┬─────────┘           │               │
                │                     │               │
         ┌──────▼──────┐              │               │

- Обновление параметров: каждые ~15 кадров (~500 мс при 30 fps)
- EMA-сглаживание: `new = prev × 0.85 + raw × 0.15`
- Диапазон всех параметров: **0–100**

## 2. Три конвейера данных

### Конвейер A: rPPG (пульс через камеру)

**Файлы**: `camera.js` → `rppg.js` → `vitals.js`

**Принцип**: кожа лица меняет цвет с каждым ударом сердца (приток крови). Камера улавливает эти микроизменения.

**Шаг 1 — Захват ROI (camera.js)**
- Область лба: центральная 1/3 ширины, верхние 15–30% высоты
- `getForeheadROI(w, h)` → `{x: w/3, y: h×0.15, w: w/3, h: h×0.15}`
- `extractROIPixels(ctx, roi)` → средние R, G, B по всем пикселям ROI

**Шаг 2 — CHROM-алгоритм (rppg.js)**

Алгоритм De Haan & Jeanne (2013) — хроминантный метод:

```
S1 = R/μR - G/μG
S2 = R/μR + G/μG - 2·B/μB
α  = σ(S1) / σ(S2)
H  = S1 + α·S2         ← пульсовой сигнал
```

- Буфер: 256 кадров (~8.5 сек при 30 fps)
- Минимум для первого результата: 64 кадра (~2 сек)
- Старые кадры вытесняются (FIFO)

**Шаг 3 — ЧСС (rppg.js)**
- DFT (N=256, O(N²)) → амплитудный спектр
- Поиск пика в диапазоне 40–180 BPM (0.67–3.0 Гц)
- `freqHz = peakBin × 30 / N`, `BPM = freqHz × 60`

**Шаг 4 — HRV-метрики (vitals.js)**

Из пульсового сигнала извлекаются пики → межпульсовые интервалы (IBI):

| Метрика | Описание | Формула |
|---------|----------|---------|
| RMSSD | Корень среднего квадрата последовательных разностей IBI | `√(Σ(IBI[i]-IBI[i-1])² / (N-1))` |
| SDNN | Стандартное отклонение IBI | `√(Σ(IBI-mean)² / N)` |
| pNN50 | % интервалов с разностью > 50 мс | `count(|ΔIBI| > 50) / N × 100` |
| LF/HF | Баланс симп./парасимп. НС | Мощность 0.04–0.15 Гц / 0.15–0.40 Гц |
| SI Баевского | Индекс напряжения ВНС | `AMo / (2 × Mo × MxDMn)` |
| Когерентность | Доминанта ~0.1 Гц ритма (HeartMath) | `peakPower / totalPower × 300`, 0–100 |
| Частота дыхания | Из RSA (zero-crossings IBI) | Вдохов/мин, диапазон 8–25 |

### Конвейер B: Vibraimage (микродвижения)

**Файл**: `vibraimage.js`

**Принцип**: голова человека непрерывно совершает микродвижения (тремор) — результат работы мышц и нервной системы. По методологии Виктора Минкина.

**ROI**: 20–80% по X, 10–80% по Y (область лица)

**Метод**:
1. Каждый кадр преобразуется в яркость: `Y = 0.299R + 0.587G + 0.114B`
2. Вычисляется попиксельная разность яркости между текущим и предыдущим кадром
3. Левая/правая половины анализируются отдельно (для симметрии)

**Буфер**: 128 кадров (~4 сек)

| Метрика | Описание | Формула |
|---------|----------|---------|
| Amplitude | Средняя величина микродвижений, 0–100 | `min(100, meanDiff / 4 × 100)` |
| Frequency | Частота осцилляций (zero-crossing rate), 0–100 | `oscillations/sec / 12 × 100` |
| Symmetry | Баланс L/R движений, 0–100 (50=идеал) | `max(0, (1 - asymmetry×2)) × 100` |
| Entropy | Регулярность паттернов (Шеннон), 0–100 | `entropySum / log2(10) × 100` |

### Конвейер C: Голос

**Файл**: `voice.js`

**Метод**: Web Audio API → `AnalyserNode` (FFT size 4096)

| Метрика | Описание | Формула |
|---------|----------|---------|
| Pitch (F0) | Основной тон через автокорреляцию | Диапазон 60–500 Гц, порог RMS > 0.01 |
| Jitter | Вариация периода pitch (%) | `Σ|P[i]-P[i-1]| / (N-1) / meanP × 100` |
| Shimmer | Вариация амплитуды (%) | `Σ|A[i]-A[i-1]| / (N-1) / meanA × 100` |
| HNR | Гармоники vs. шум (дБ) | `10 × log10(harmEnergy / noiseEnergy)` |
| RMS | Среднеквадратичная громкость | `√(Σsample² / N)` |
| Spectral Centroid | "Яркость" звучания (Гц) | `Σ(freq × mag) / Σmag` |
| Formants | Резонансные частоты F1/F2/F3 | Пики в сглаженном спектре 200–4000 Гц |
| VoiceBio Center | Энергетический центр 0–6 | Pitch → нота C/D/E/F/G/A/B → индекс |

**VoiceBio маппинг** (base octave C3=128 Гц):

| Нота | Центр (Гц) | Индекс | Параметр |
|------|-----------|--------|----------|
| C | 131.8 | 0 | Стабильность |
| D | 147.9 | 1 | Поток |
| E | 166.0 | 2 | Энергия |
| F | 175.9 | 3 | Резонанс |
| G | 197.5 | 4 | Вибрация |
| A | 221.7 | 5 | Ясность |
| B | 248.8 | 6 | Целостность |

Высокие октавы складываются вниз (`while f > 256: f /= 2`).

## 3. Детальное описание 7 параметров

### 3.1 Стабильность (Stability)

**Цвет**: `#ff3366` | **Частота**: 396 Hz

**Формула**:
```
base = HRV_score × 0.7 + SDNN_score × 0.3
    (где HRV_score = 100 - |RMSSD - 50| × 2, оптимум RMSSD ≈ 50 мс)
    (где SDNN_score = 100 - |SDNN - 75| × 2, оптимум SDNN ≈ 75 мс)
- stress_penalty = max(0, min(30, (SI - 150) × 0.1))
    (штраф при индексе Баевского > 150)
+ vibraimage.symmetry × 0.2
    (бонус за симметрию микродвижений лица)
```

**Входные метрики**:
| Метрика | Оптимум | Источник |
|---------|---------|----------|
| RMSSD | ~50 мс | rPPG → HRV |
| SDNN | ~75 мс | rPPG → HRV |
| Stress Index | < 150 | rPPG → Баевский |
| Vibraimage symmetry | высокая (100) | Камера → vibraimage |

**Fallback**: без камеры / без данных → 50 (нейтраль)

**Примеры**:
- Спокойное ровное дыхание → RMSSD ~50, SI < 100, симметрия высокая → Стабильность ~80
- Стресс → RMSSD скачет, SI > 200 → штраф 5+ пунктов → Стабильность ~45
- Без камеры, только голос → Стабильность = 50 (нейтраль, так как все входы от камеры)

---

### 3.2 Поток (Flow)

**Цвет**: `#ff7a2e` | **Частота**: 417 Hz

**Формула**:
```
base = hrSmoothed_norm × 0.7 + pNN50_score × 0.3
    (hrSmoothed_norm = norm(hrSmoothed, 0, 100))
    (pNN50_score = 100 - |pNN50 - 20| × 3, оптимум pNN50 ≈ 20%)
+ (100 - vibraimage.entropy) × 0.2
    (низкая энтропия = регулярные паттерны = лучший поток)
```

**Входные метрики**:
| Метрика | Оптимум | Источник |
|---------|---------|----------|
| hrSmoothed | ~50–70 (после EMA) | rPPG → HR → EMA |
| pNN50 | ~20% | rPPG → HRV |
| Vibraimage entropy | низкая (0) | Камера → vibraimage |

**Fallback**: без камеры → 50

**Примеры**:
- Ритмичное дыхание, ровный пульс, минимум лишних движений → Поток ~75
- Нервная суета, хаотичные микродвижения → энтропия высокая → Поток ~40

---

### 3.3 Энергия (Energy)

**Цвет**: `#ffcc00` | **Частота**: 528 Hz

**Формула**:
```
HR_zone = 100 - |HR - 70| × 2.5     (оптимум HR ≈ 70 уд/мин)
volume  = norm(RMS, 0, 0.3)           (громкость голоса)

Если vibraimage доступен:
  base = HR_zone × 0.5 + volume × 0.3 + vibra_amp_zone × 0.2
  (vibra_amp_zone = 100 - |amplitude - 40| × 2.5, оптимум ~40)
Иначе:
  base = HR_zone × 0.6 + volume × 0.4
```

**Входные метрики**:
| Метрика | Оптимум | Источник |
|---------|---------|----------|
| HR | ~70 уд/мин | rPPG |
| Voice RMS | 0–0.3 (громче = выше) | Микрофон |
| Vibraimage amplitude | ~40 (умеренное) | Камера → vibraimage |

**Fallback**: без микрофона → RMS = 0, volume = 50 (neutral norm). Без камеры → HR нет → HR_zone = 50

**Примеры**:
- HR 72, громкий голос, умеренный тремор → Энергия ~70
- HR 110 (тахикардия) → HR_zone резко падает → Энергия ~35
- Молчание + нормальный пульс → volume = 0 → Энергия опирается больше на HR

---

### 3.4 Резонанс (Resonance)

**Цвет**: `#00ff99` | **Частота**: 639 Hz

**Формула**:
```
base = coherence × 0.7 + LF/HF_score × 0.3
    (coherence: 0–100, по методологии HeartMath)
    (LF/HF_score = 100 - |LF/HF - 1.5| × 40, оптимум LF/HF ≈ 1.5)
```

**Входные метрики**:
| Метрика | Оптимум | Источник |
|---------|---------|----------|
| Coherence | высокая (100) | rPPG → HRV спектр |
| LF/HF | ~1.0–2.0 | rPPG → HRV спектр |

**Fallback**: без камеры → coherence = 50, lfhf = null → Резонанс = 50

**Примеры**:
- Медленное глубокое дыхание (6 вд/мин) → когерентность ~80, LF/HF ~1.2 → Резонанс ~75
- Стресс → LF/HF > 4 (симпатикотония) → LF/HF_score ≈ 0 → Резонанс ~50

---

### 3.5 Вибрация (Vibration)

**Цвет**: `#00ccff` | **Частота**: 741 Hz

**Формула**:
```
pitch_norm    = norm(pitch, 80, 300)           (0–100)
centroid_norm = norm(spectralCentroid, 500, 4000) (0–100)

Если vibraimage доступен:
  base = pitch_norm × 0.4 + centroid_norm × 0.4 + vibra.frequency × 0.2
Иначе:
  base = pitch_norm × 0.5 + centroid_norm × 0.5
```

**Входные метрики**:
| Метрика | Оптимум | Источник |
|---------|---------|----------|
| Pitch | 80–300 Гц | Микрофон |
| Spectral centroid | 500–4000 Гц | Микрофон |
| Vibraimage frequency | высокая (100) | Камера → vibraimage |

**Fallback**: без микрофона → pitch = null, centroid = null → оба = 50. Без камеры → vibra.frequency отсутствует

**Примеры**:
- Говорите громко, богатый тембр → pitch ~180, centroid ~2000 → Вибрация ~55
- Молчание → pitch/centroid = null = 50, vibra.frequency добавляет → Вибрация ~50

---

### 3.6 Ясность (Clarity)

**Цвет**: `#6677ff` | **Частота**: 852 Hz

**Формула**:
```
HNR_score    = norm(HNR, 0, 25)               (0–100)
jitter_score = norm(jitter, 0, 5, invert)      (0–100, меньше jitter = лучше)
formant_score = (F1_ok ? 75 : 30)×0.5 + (F2_ok ? 75 : 30)×0.5
    (F1_ok: 250–900 Гц, F2_ok: 800–2800 Гц)

Если vibraimage доступен:
  calm_score = max(0, 100 - amplitude)
  base = HNR_score × 0.4 + jitter_score × 0.25 + formant_score × 0.15 + calm_score × 0.2
Иначе:
  base = HNR_score × 0.5 + jitter_score × 0.3 + formant_score × 0.2
```

**Входные метрики**:
| Метрика | Оптимум | Источник |
|---------|---------|----------|
| HNR | высокий (~25 дБ) | Микрофон |
| Jitter | низкий (~0%) | Микрофон |
| Formants F1, F2 | F1: 250–900, F2: 800–2800 | Микрофон |
| Vibraimage amplitude | низкая (0 = покой) | Камера → vibraimage |

**Fallback**: без микрофона → все голосовые = null = 50. Без камеры → amplitude отсутствует

**Примеры**:
- Чистый ровный голос, тело неподвижно → HNR ~20, jitter ~1% → Ясность ~70
- Хриплый голос + суета → HNR ~5, jitter ~4%, amplitude высокая → Ясность ~30

---

### 3.7 Целостность (Integrity)

**Цвет**: `#bb44ff` | **Частота**: 963 Hz

**Формула**:
```
params = [Стабильность, Поток, Энергия, Резонанс, Вибрация, Ясность]
mean = среднее(params)
variance = Σ(param - mean)² / 6
integrity = max(0, 100 - √variance × 3)
```

**Входные метрики**: все 6 предыдущих параметров

**Fallback**: если все параметры = 50, дисперсия = 0 → Целостность = 100

**Примеры**:
- Все параметры в районе 60–70 → дисперсия мала → Целостность ~85
- Стабильность 90, Энергия 20, остальные ~50 → разброс большой → Целостность ~40

---

### 3.8 Светимость (Luminosity)

Не отдельный параметр, а средневзвешенное всех 7:
```
luminosity = round(Σ(все 7 параметров) / 7)
```

## 4. EMA-сглаживание

```javascript
smoothed[key] = prev[key] × (1 - α) + raw[key] × α    // α = 0.15
```

### Свойства

- Новому значению даётся вес **15%**, старому — **85%**
- Время выхода на 90% нового значения: ~15 итераций = ~7.5 сек
- Переход с 80 до 55: занимает ~6 секунд (25 пунктов × 15% за итерацию)

### Известные проблемы

1. **Инерция при уходе из кадра**: когда лицо уходит из кадра, rPPG и vibraimage продолжают читать фон (стену), буферы не очищаются, и EMA с α=0.15 очень медленно обновляет значения. Параметры "застревают" на старых значениях.

2. **breathingRate вычисляется, но не используется**: `vitals.js` рассчитывает частоту дыхания из RSA, значение передаётся в `mapToBiofield()`, но `biofield.js` его игнорирует.

3. **Нет детекции потери лица**: `aura.js` имеет проверку `count > 20` (достаточно пикселей кожи), но не экспортирует состояние — нет возможности узнать, что лицо потеряно.
